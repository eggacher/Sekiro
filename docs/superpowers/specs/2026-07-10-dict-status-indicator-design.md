# 数据字典类型状态视觉标识设计方案

本文档旨在设计一种视觉优化方案，为数据字典页面中的字典类型添加“启用/停用”状态标识，以解决用户停用某类字典时前台列表和详情缺乏明确标志反馈的问题。

## 用户评审要点

> [!IMPORTANT]
> 1. 本变更为纯前端 UI/UX 体验改进，无须任何后端 API 重构。
> 2. 状态标签组件沿用了系统现有的 `StatusBadge` 统一样式，保证全局设计语言的一致性。

## 设计细节

修改 [dict/page.tsx](file://<PROJECT_ROOT>/apps/web/app/(dashboard)/system/dict/page.tsx) 页面。

### 1. 左侧字典类型列表项视觉增强
在 `filteredTypes.map((d) => ...)` 列表项中：
- 当该类型 `status === "disabled"` 时，对外部 `button` 容器追加 `opacity-60` 进行微灰化，代表“非活跃”。
- 将 `BookMarked` 图标的颜色设为 `text-muted-foreground`（禁用时）或 `text-primary`（启用时）。
- 在字典名称文本行右侧，利用 Flex 布局的 `justify-between` 追加一个小型红色线框 Badge `[停用]` 以引起警示。

### 2. 右侧字典详情头部增强
在右侧字典项表格上方的当前选中类型详情栏中：
- 在字典名称与字典编码的右侧，统一插入系统已有的 `<StatusBadge status={active.status} />` 控制组件，使用户在查看其包含的具体字典项时，能一目了然得知该类别的全局状态。

## 验证与测试方案

### 自动化验证
- `pnpm typecheck` 前端无编译和类型检查错误。

### 手动验证
1. 点击左侧“新增”字典类型，将状态设为“禁用”并创建。
2. 观察左侧字典类型列表：
   - 新创建的字典类型应该显示为稍微淡化的半透明状态。
   - 字典名称最右侧出现红色的 `[停用]` 标签。
3. 点击选中该停用字典：
   - 右侧详情面板头部的标题和编码右侧，应正常渲染红色线框的“停用”状态标签。
4. 将其编辑更新为“启用”：
   - 左侧变回正常亮色，且红色标签消失。
   - 右侧头部状态变成绿色的“启用”状态标签。
