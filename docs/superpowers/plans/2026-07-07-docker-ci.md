# Story #26: Docker / CI 基础镜像与 GitHub Actions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 Sekiro 后端 API 和前端 Web 构建生产级 Docker 镜像，扩展 docker-compose 编排，新增 `/health` 健康检查端点，实现单台服务器 `docker compose` 一键部署。

**Architecture:** 采用独立服务镜像策略：API 与 Web 各一个多阶段 Dockerfile；使用 `node:20-alpine` 基础镜像；Next.js 启用 standalone 输出；通过单一 `docker-compose.yml` 同时支持本地基础设施和完整服务启动；API 暴露 `/health` 端点供 Docker HEALTHCHECK 使用。

**Tech Stack:** Docker、Docker Compose、Node.js 20、pnpm 9、NestJS、Next.js 14、Prisma、Alpine Linux

## Global Constraints

- Node.js >= 18.17.0，pnpm >= 9（`packageManager: pnpm@9.15.9`）
- 基础镜像统一使用 `node:20-alpine`
- Dockerfile 中必须使用 `--frozen-lockfile` 保证依赖可复现
- 不引入 GitHub Actions CI，不引入 K8s/Helm/反向代理/SSL
- 保持本地开发原命令不变：`pnpm docker:up` 只起 postgres + redis
- 所有代码修改必须通过 `pnpm typecheck` 和 `pnpm --filter @sekiro/api test`
- 工作目录为 isolated worktree：`.worktrees/feature/docker-ci`

**Files:**

**Interfaces:**
- Consumes: NestJS `@Controller`, `@Get`
- Produces: `GET /health` 返回 `{ status: "ok", timestamp: string }`

- [ ] **Step 1: 创建 HealthController**

```ts
import { Controller, Get } from "@nestjs/common";

@Controller()
export class HealthController {
  @Get("/health")
  health() {

      status: "ok",
      timestamp: new Date().toISOString(),
```

## Task 3: 编写 API Dockerfile

**Files:**
- Create: `apps/api/Dockerfile`

**Interfaces:**
- Consumes: pnpm workspace monorepo root files, `apps/api/package.json`, `packages/shared/package.json`
- Produces: Production image exposing port 3001, running `node dist/main.js`

- [ ] **Step 1: 创建 Dockerfile**

```dockerfile
FROM node:20-alpine AS base
ENV PNPM_HOME="/pnpm"
ENV PATH="$PNPM_HOME:$PATH"
RUN corepack enable && corepack prepare pnpm@9.15.9 --activate

FROM base AS deps
WORKDIR /app
COPY pnpm-workspace.yaml package.json pnpm-lock.yaml ./
COPY apps/api/package.json ./apps/api/
COPY apps/web/package.json ./apps/web/
COPY packages/shared/package.json ./packages/shared/
RUN --mount=type=cache,id=pnpm,target=/pnpm/store pnpm install --frozen-lockfile

FROM deps AS builder
WORKDIR /app
COPY . .
RUN pnpm --filter @sekiro/api prisma:generate
RUN pnpm --filter @sekiro/api build

FROM node:20-alpine AS runner
RUN apk add --no-cache dumb-init
WORKDIR /app
ENV NODE_ENV=production
ENV PORT=3001
COPY --from=builder /app/apps/api/package.json ./
COPY --from=builder /app/apps/api/dist ./dist
COPY --from=builder /app/apps/api/prisma ./prisma
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/apps/api/node_modules ./node_modules/@sekiro/api/node_modules 2>/dev/null || true
EXPOSE 3001
HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
  CMD wget -qO- http://localhost:3001/health || exit 1
USER node
CMD ["dumb-init", "node", "dist/main.js"]
```

- [ ] **Step 2: 构建镜像并验证**

```bash
docker build -t sekiro-api:latest -f apps/api/Dockerfile .
```

Expected: 构建成功，无 ERROR 退出。

- [ ] **Step 3: 本地运行 API 容器验证**

```bash
docker run -d --name sekiro-api-test \
  -p 3002:3001 \
  -e DATABASE_URL="postgresql://sekiro:sekiro123@host.docker.internal:5432/sekiro?schema=public" \
  -e DIRECT_DATABASE_URL="postgresql://sekiro:sekiro123@host.docker.internal:5432/sekiro?schema=public" \
  -e REDIS_URL="redis://host.docker.internal:6379" \
  sekiro-api:latest
```

> 注：macOS 使用 `host.docker.internal` 访问宿主机服务；Linux 需加 `--add-host=host.docker.internal:host-gateway`。

## Task 5: 扩展 docker-compose.yml

**Files:**
- Modify: `docker-compose.yml`

**Interfaces:**
- Consumes: `sekiro-api:latest` / `sekiro-web:latest` 镜像
- Produces: 完整服务编排，支持 `docker compose up -d postgres redis` 和 `docker compose up -d`

- [ ] **Step 1: 重写 docker-compose.yml**

```yaml
# 开发环境依赖 + 完整服务一键启动
# 本地开发只起基础设施：docker compose up -d postgres redis
# 完整启动：docker compose up -d
version: "3.9"

services:
  postgres:
    image: postgres:16-alpine
    container_name: sekiro-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: sekiro
      POSTGRES_PASSWORD: sekiro123
      POSTGRES_DB: sekiro
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U sekiro -d sekiro"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: sekiro-redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  api:
    build:
      context: .
      dockerfile: apps/api/Dockerfile
    image: sekiro-api:latest
    container_name: sekiro-api
    restart: unless-stopped
    environment:
      DATABASE_URL: "postgresql://sekiro:sekiro123@postgres:5432/sekiro?schema=public"
      DIRECT_DATABASE_URL: "postgresql://sekiro:sekiro123@postgres:5432/sekiro?schema=public"
      REDIS_URL: "redis://redis:6379"
      NODE_ENV: production
      PORT: 3001
    ports:
      - "3001:3001"
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy

  web:
    build:
      context: .
      dockerfile: apps/web/Dockerfile
    image: sekiro-web:latest
    container_name: sekiro-web
    restart: unless-stopped
    environment:
      API_URL: "http://api:3001"
      NODE_ENV: production
      PORT: 3000
    ports:
      - "3000:3000"
    depends_on:
      - api

volumes:
  postgres_data:
  redis_data:
```
