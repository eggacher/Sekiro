# Story #28: i18n 国际化 + 主题切换 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 Sekiro 前端硬编码中文全面替换为 i18n key，重构字典为按模块拆分结构，并验证修复主题切换功能。

**Architecture:** 保留现有 client-side React Context 的 i18n 方案，将 `dictionaries/zh.ts` / `en.ts` 拆分为 `dictionaries/zh/` 与 `dictionaries/en/` 下的模块文件；使用 `useTranslation()` hook 在 Client Components 中替换硬编码文本；主题切换继续使用 `next-themes` 与 CSS 变量，只修复发现的暗色模式问题。

**Tech Stack:** Next.js 14, React, TypeScript, next-themes, Tailwind CSS, shadcn/ui, recharts

## Global Constraints

- 所有新增/修改的组件必须保持 `"use client"` 如果使用了 `useTranslation`
- `TranslationKey` 类型必须由 `typeof zh` 推导，保持类型安全
- 不引入新依赖（已有 `next-themes`）
- 不改动后端 API 错误消息
- 每次提交前必须跑 `pnpm typecheck`
- 每个页面/组件任务完成后建议切到浏览器验证英文显示
- 工作目录为 isolated worktree：`.worktrees/feature/i18n-themes`
