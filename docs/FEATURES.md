# 脚手架项目功能清单（带管理后台）

> 本文档梳理一个"带管理后台的脚手架项目"应具备的功能与特性，按 **优先级分层**（P0 核心 / P1 推荐 / P2 增强）组织，便于做取舍与排期。
>
> - **P0 核心**：最小可用版本（MVP）必备，缺一不可。
> - **P1 推荐**：成熟脚手架的标配，提升易用性与完整性。
> - **P2 增强**：锦上添花，面向生产级或大型团队场景。

## 一、认证与授权（Auth）

### P0 核心
- [ ] 登录 / 登出（账号密码）
- [ ] 会话管理（JWT 或 Session + Cookie）
- [ ] Token 自动刷新 / 无感续期
- [ ] 路由守卫（未登录跳转、登录态失效处理）
- [ ] RBAC 角色权限模型（用户-角色-权限）
- [ ] 接口级权限校验（后端中间件）
- [ ] 页面级 / 按钮级权限控制（前端指令或组件，如 `v-permission`）

### P1 推荐
- [ ] 动态路由 & 动态菜单（根据权限下发）
- [ ] 修改密码、密码强度校验
- [ ] 验证码登录（图形 / 滑块 / 短信 / 邮箱）
- [ ] 第三方登录（GitHub / Google / 微信等 OAuth）
- [ ] 多因素认证（MFA / TOTP）

### P2 增强
- [ ] 单点登录（SSO / SAML / OIDC）
- [ ] 设备管理 & 异地登录提醒
- [ ] 操作锁定（多次失败锁定账号）

## 二、用户与组织（User & Org）

### P0 核心
- [ ] 用户管理（增删改查、批量操作、状态启停）
- [ ] 角色管理（角色 CRUD、权限分配）
- [ ] 菜单 / 权限项管理
- [ ] 个人中心（查看资料、修改信息）

### P1 推荐
- [ ] 部门 / 组织架构管理（树形）
- [ ] 数据权限（按部门范围控制数据可见性）
- [ ] 用户导入 / 导出（Excel）
- [ ] 岗位管理

---

## 三、布局与 UI（Layout）

### P0 核心
- [ ] 经典后台布局（侧边栏 + 顶栏 + 内容区）
- [ ] 响应式 / 移动端适配
- [ ] 面包屑导航
- [ ] 折叠 / 展开侧边栏

### P1 推荐
- [ ] 多页签（Tabs，类似浏览器标签）
- [ ] 主题切换（亮色 / 暗色 / 跟随系统）
- [ ] 主题色定制
- [ ] 全屏、水印
- [ ] 国际化 i18n（至少中英文）

### P2 增强
- [ ] 多布局模式切换（侧边 / 顶部 / 混合）
- [ ] 自定义拖拽仪表盘 Dashboard

---

## 四、通用业务组件（复用能力，脚手架的核心价值）

### P0 核心
- [ ] 标准 CRUD 表格（搜索 + 表格 + 分页 + 新增/编辑弹窗）
- [ ] 表单封装（校验、布局、联动）
- [ ] 请求封装（统一拦截、错误处理、loading）
- [ ] 消息提示（toast / 确认框 / 通知）

### P1 推荐
- [ ] 文件 / 图片上传组件（支持 OSS / S3 / 本地）
- [ ] 富文本编辑器
- [ ] 字典 / 枚举下拉（结合数据字典）
- [ ] 图标选择器
- [ ] 时间 / 日期范围选择
- [ ] 导入导出工具

### P2 增强
- [ ] 低代码表单 / 表单设计器
- [ ] 可视化图表（ECharts 封装）
- [ ] Markdown 编辑器

## 五、系统配置与字典（System）

### P0 核心
- [ ] 环境变量管理（dev / test / prod 多环境）
- [ ] 统一的应用配置文件

### P1 推荐
- [ ] 数据字典管理（性别、状态等枚举集中维护）
- [ ] 系统参数配置（运行时可改的配置项）
- [ ] 短信 / 邮件模板管理

## 六、系统监控与日志（Monitor & Log）

### P1 推荐
- [ ] 操作日志（记录增删改行为，可查询审计）
- [ ] 登录日志（登录时间、IP、设备、地理位置）
- [ ] 在线用户管理（强制下线）
- [ ] 服务器监控（CPU / 内存 / 磁盘 / JVM 或进程）

### P2 增强
- [ ] 错误日志 / 异常上报
- [ ] 接口耗时与慢查询监控
- [ ] 定时任务管理（可视化创建、启停、查看执行日志）
- [ ] 消息队列监控

## 七、开发体验（DX，决定脚手架好不好用）

### P0 核心
- [ ] 清晰的项目目录约定
- [ ] API 请求层统一封装 + 类型定义（如 TS 接口类型生成）
- [ ] 环境变量类型安全
- [ ] README + 启动文档

### P1 推荐
- [ ] ESLint + Prettier + 提交规范（Commitizen / commitlint）
- [ ] Git Hooks（husky + lint-staged）
- [ ] 路径别名（`@/`）
- [ ] Mock 数据方案
- [ ] ~~代码生成器（根据数据库表自动生成 CRUD 前后端代码）⭐ 脚手架杀手锏~~
  - **已放弃**：Vibe Coding / AI 辅助开发时代，手动维护代码生成器 ROI 低，由 AI 直接生成更灵活。

### P2 增强
- [ ] 单元测试框架集成（Vitest / Jest）
- [ ] E2E 测试（Playwright / Cypress）
- [ ] 组件文档（Storybook）
- [ ] 性能分析工具

## 八、后端基础设施（如果脚手架包含后端）

### P0 核心
- [ ] ORM 集成 + 迁移脚本（如 Prisma / TypeORM / MyBatis / GORM）
- [ ] 统一响应结构（`{ code, message, data }`）
- [ ] 全局异常处理
- [ ] 参数校验（DTO 校验）
- [ ] CORS 配置
- [ ] 数据库种子数据（seed，开箱即用有测试账号）

### P1 推荐
- [ ] Redis 缓存集成
- [ ] 文件存储抽象层（本地 / OSS / S3 可切换）
- [ ] 定时任务框架
- [ ] WebSocket 实时通信（可选）
- [ ] 接口限流 / 防刷
- [ ] 接口文档（Swagger / OpenAPI 自动生成）

### P2 增强
- [ ] 消息队列集成
- [ ] 分布式锁
- [ ] 搜索引擎（Elasticsearch / Meilisearch）

## 九、安全（Security）

### P0 核心
- [ ] 密码加密存储（bcrypt / argon2）
- [ ] XSS / CSRF 防护
- [ ] SQL 注入防护（参数化查询）
- [ ] 敏感信息脱敏（日志中的密码、token）

### P1 推荐
- [ ] 接口限流（Rate Limit）
- [ ] 安全响应头（Helmet 类）
- [ ] 文件上传校验（类型、大小、内容）
- [ ] 密钥 / 配置加密管理

---

## 十、部署与运维（DevOps）

### P1 推荐
- [ ] Dockerfile + docker-compose（一键起依赖：DB / Redis）
- [ ] 健康检查接口（`/health`）
- [ ] CI/CD 配置示例（GitHub Actions / GitLab CI）
- [ ] 构建产物优化（压缩、分包、CDN）

### P2 增强
- [ ] K8s 部署清单
- [ ] 日志收集方案
- [ ] 监控告警接入（Prometheus / Grafana）

## 建议的最小可用版本（MVP / P0 合集）

如果时间有限，先做这一套就能称之为"可用的后台脚手架"：

> **登录鉴权 + RBAC 权限 + 动态菜单 + 用户/角色/菜单管理 + 标准 CRUD 表格组件 + 请求封装 + 主题/i18n + ORM + 种子数据 + Docker 一键启动 + README**
